# 资源系统 (Resource System)

> **Status**: Designed (pending review)
> **Author**: user + game-designer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 收集满足（资源积累的满足感）

## Overview

资源系统管理玩家的金币获取、消耗和余额。金币是游戏的主要货币，用于购买单位和可能的升级。系统确保资源变更有清晰的来源和去向，防止负余额。

## Player Fantasy

这个系统服务于「收集满足」支柱。玩家应该能够清晰地看到金币的来源和用途，积累财富带来成就感，消费带来收获的期待。

## Detailed Design

### Core Rules

#### 1. 资源类型

| 资源 | 描述 | MVP 状态 |
|------|------|----------|
| **Gold（金币）** | 主要货币 | ✅ 使用 |
| Gems（宝石） | 高级货币 | ❌ 后期 |

#### 2. 金币来源

| 来源 | 数量 | 说明 |
|------|------|------|
| 战斗胜利 | 关卡奖励 | 由关卡定义 |
| 每日登录 | 固定数量 | 后期功能 |
| 成就奖励 | 固定数量 | 后期功能 |

#### 3. 金币消耗

| 消耗 | 数量 | 说明 |
|------|------|------|
| 购买单位 | 单位价格 | 由单位定义 |
| 升级单位 | 升级价格 | 后期功能 |

#### 4. 余额规则

- 金币余额不能为负
- 金币有上限（MAX_GOLD）
- 消费前检查余额

### States and Transitions

**无状态机**。资源系统是简单的数值管理。

### Interactions with Other Systems

| 系统 | 接口 | 用途 |
|------|------|------|
| **玩家数据** | gold 字段 | 存储余额 |
| **战斗系统** | add_gold(amount) | 胜利奖励 |
| **商店系统** | spend_gold(amount) → bool | 购买消耗 |

## Formulas

### 余额计算

```
new_balance = clamp(current_gold + delta, 0, MAX_GOLD)
```

### 交易验证

```
can_afford(price) = current_gold >= price
```

## Edge Cases

| 情况 | 处理方式 |
|------|----------|
| 金币为 0 时消费 | 拒绝交易 |
| 获得金币超过上限 | 截断到上限 |
| 负数金币输入 | 拒绝，日志警告 |

## Dependencies

| 系统 | 依赖内容 | 状态 |
