# 单位数据定义 (Unit Data Definition)

> **Status**: Designed (pending review)
> **Author**: user + game-designer
> **Last Updated**: 2026-03-23
> **Implements Pillar**: 收集满足（每个单位都是宝藏）

## Overview

单位数据定义系统定义了游戏中所有单位的属性、技能、职业和稀有度。它是游戏的数据基础，所有战斗、收集和商店系统都依赖它来理解"什么是单位"。玩家不直接操作这个系统，但他们收集的每一个单位、战斗中的每一次攻击、商店里的每一次购买，都由这里定义的数据驱动。

## Player Fantasy

这个系统服务于「收集满足」支柱。玩家应该感觉到每个单位都是独特的、有价值的宝藏——而不是仅仅是数值差异的容器。当玩家获得一个新单位时，他们应该能立即识别出它的独特性：这是一个法师而不是战士，它有独特的技能，它属于某个稀有度等级。这种独特性创造了收集的动力，因为每个单位都提供了不同的战术可能性。

## Detailed Design

### Core Rules

#### 1. 单位属性

每个单位拥有以下核心属性：

| 属性 | 描述 | 数据类型 | 典型范围 |
|------|------|----------|----------|
| **hp** | 生命值 | int | 100-1000 |
| **attack** | 攻击力 | int | 10-100 |
| **attack_speed** | 攻击速度（次/秒） | float | 0.5-2.0 |
| **attack_range** | 攻击范围（格子数） | int | 1-4 |
| **armor** | 护甲（减少伤害） | int | 0-50 |
| **move_speed** | 移动速度（格/秒） | float | 0.0-2.0 |

#### 2. 单位元数据

| 属性 | 描述 | 可选值 |
|------|------|--------|
| **id** | 唯一标识符 | string |
| **name** | 显示名称 | string |
| **class_type** | 职业类型 | Warrior, Archer, Mage, Knight, Healer |
| **rarity** | 稀有度 | Common, Rare, Epic, Legendary |
| **skill** | 技能对象引用 | Skill |
| **price** | 商店价格（金币） | int |

#### 3. 单位大小

- 所有单位占用 **1 格**（1x1）
- 简化布局逻辑，降低策略复杂度

#### 4. 职业类型定义

| 职业 | 定位 | 特点 |
|------|------|------|
| **Warrior（战士）** | 近战坦克 | 高 HP，低攻击范围，中等攻击力 |
| **Archer（弓手）** | 远程输出 | 低 HP，高攻击范围，高攻击速度 |
| **Mage（法师）** | 法术输出 | 低 HP，中等范围，高攻击力 |
| **Knight（骑士）** | 近战输出 | 中等 HP，低范围，高攻击力 |
| **Healer（治疗）** | 辅助 | 低 HP，中等范围，治疗技能 |

#### 5. 稀有度规则

| 稀有度 | 属性加成 | 技能强度 | 价格倍率 |
|--------|----------|----------|----------|
| **Common（普通）** | 1.0x | 1.0x | 1x |
| **Rare（稀有）** | 1.2x | 1.2x | 2x |
| **Epic（史诗）** | 1.5x | 1.5x | 4x |
| **Legendary（传说）** | 2.0x | 2.0x | 10x |

#### 6. 技能规则

- 每个单位拥有 **1 个技能**
- 技能可以是被动（持续生效）或主动（冷却触发）
- 技能定义在独立的技能系统中，单位数据通过引用关联

#### 7. 数据不可变性

- 单位定义数据是**静态的、不可变的**
- 玩家拥有的单位实例包含：单位定义引用 + 当前等级
- 等级提升在玩家数据中处理，不修改单位定义

### States and Transitions

**此系统无运行时状态转换。**

单位数据定义是一个静态数据配置系统。所有单位定义在游戏启动时加载，运行期间不会改变。

状态相关的逻辑在其他系统中处理：
- **单位实例状态**（如当前 HP、存活/死亡）在战斗系统中管理
- **玩家拥有状态**（如是否解锁、当前等级）在玩家数据中管理

### Interactions with Other Systems

#### 输出接口（其他系统从此获取数据）

| 目标系统 | 输出数据 | 用途 |
|----------|----------|------|
| **伤害计算系统** | attack, armor, rarity_multiplier | 计算伤害输出和减免 |
| **目标选择 AI** | class_type, attack_range, move_speed | 决定目标优先级和移动行为 |
| **网格布局系统** | id, name, class_type | 显示单位信息，验证放置规则 |
| **克制系统** | class_type | 查找克制关系 |
| **单位协同系统** | id, class_type, skill | 检测协同组合 |
| **商店系统** | id, name, rarity, price | 显示商品，处理购买 |
| **敌人系统** | 全部属性 | 敌人也使用单位数据定义 |
| **布局编辑 UI** | id, name, class_type, rarity | 显示单位列表和信息 |
| **商店 UI** | id, name, rarity, price, class_type | 显示商店商品 |

#### 输入接口（此系统从其他系统获取数据）

| 来源系统 | 输入数据 | 用途 |
|----------|----------|------|
| **技能系统** | skill 定义 | 关联单位与技能 |

#### 接口规范

```gdscript
# 单位定义接口（GDScript 示例）
class_name UnitDefinition extends Resource

@export var id: String
@export var name: String
@export var class_type: Global.ClassType  # Enum
@export var rarity: Global.Rarity         # Enum
@export var hp: int
@export var attack: int
@export var attack_speed: float
@export var attack_range: int
@export var armor: int
@export var move_speed: float
@export var skill: SkillDefinition        # Resource reference
@export var price: int

# 计算属性（根据稀有度加成）
func get_effective_hp() -> int:
    return int(hp * Global.RARITY_MULTIPLIERS[rarity])

func get_effective_attack() -> int:
    return int(attack * Global.RARITY_MULTIPLIERS[rarity])
```

## Formulas

### 稀有度加成公式

**最终属性 = 基础属性 × 稀有度系数**

```
effective_hp = base_hp × RARITY_MULTIPLIER[rarity]
effective_attack = base_attack × RARITY_MULTIPLIER[rarity]
effective_armor = base_armor × RARITY_MULTIPLIER[rarity]
```

#### 稀有度系数表

| 稀有度 | 系数 | 变量名 |
|--------|------|--------|
| Common | 1.0 | `RARITY_COMMON` |
| Rare | 1.2 | `RARITY_RARE` |
| Epic | 1.5 | `RARITY_EPIC` |
| Legendary | 2.0 | `RARITY_LEGENDARY` |

### 商店价格公式

**价格 = 基础价格 × 稀有度价格倍率**

```
effective_price = base_price × RARITY_PRICE_MULTIPLIER[rarity]
```

| 稀有度 | 价格倍率 |
|--------|----------|
| Common | 1x |
| Rare | 2x |
| Epic | 4x |
| Legendary | 10x |

### 攻击速度转换

**攻击间隔（秒） = 1 / 攻击速度**

```
attack_interval = 1.0 / attack_speed
```

示例：
- 攻击速度 0.5 → 攻击间隔 2.0 秒
- 攻击速度 1.0 → 攻击间隔 1.0 秒
- 攻击速度 2.0 → 攻击间隔 0.5 秒

### 属性范围约束

| 属性 | 最小值 | 最大值 | 说明 |
|------|--------|--------|------|
| hp | 1 | 9999 | 防止零血量单位 |
| attack | 0 | 999 | 0 攻击单位可用于特殊用途 |
| attack_speed | 0.1 | 5.0 | 防止过快或过慢 |
| attack_range | 0 | 10 | 0 = 近战，必须贴身 |
| armor | 0 | 100 | 护甲减伤有上限 |
| move_speed | 0 | 5.0 | 0 = 不移动（塔防类单位） |

## Edge Cases

### 1. 属性边界值

| 情况 | 处理方式 |
|------|----------|
| **HP = 0 或负数** | 数据验证失败，拒绝加载该单位定义 |
| **攻击速度 = 0** | 数据验证失败，最小值为 0.1 |
| **移动速度 = 0** | 合法值，表示单位不会移动（炮塔类） |
| **攻击范围 = 0** | 合法值，表示近战必须贴身攻击 |
| **护甲 > 100** | 数据验证警告，实际效果有上限（见伤害计算系统） |

### 2. 稀有度一致性

| 情况 | 处理方式 |
|------|----------|
| **同 ID 不同稀有度** | 禁止，ID 必须全局唯一 |
| **稀有度与价格不匹配** | 数据验证警告，但不阻止加载 |
| **所有单位都是传说级** | 允许，但会破坏收集动机 |

### 3. 技能引用问题

| 情况 | 处理方式 |
|------|----------|
| **技能引用为空** | 数据验证警告，单位将没有技能效果 |
| **技能引用不存在** | 数据验证失败，拒绝加载该单位定义 |
| **技能被删除** | 启动时报错，单位定义失效 |

### 4. 职业类型未知

| 情况 | 处理方式 |
|------|----------|
| **职业类型不在枚举中** | 数据验证失败，拒绝加载 |
| **克制关系缺失该职业** | 视为无克制关系，日志警告 |

### 5. 数据加载顺序

| 情况 | 处理方式 |
|------|----------|
| **技能数据先于单位加载** | 正常，单位引用技能 |
| **单位数据先于技能加载** | 引用暂时为空，技能加载后解析 |
| **循环引用** | 不可能，技能不引用单位定义 |

### 6. 热更新/补丁

| 情况 | 处理方式 |
|------|----------|
| **修改现有单位属性** | 允许，影响所有玩家的该单位 |
| **删除单位定义** | 禁止，已拥有该单位的玩家数据会出错 |
| **添加新单位** | 允许，需要更新版本号 |

## Dependencies

### 上游依赖（此系统依赖的系统）

**无。** 此系统是 Foundation 层，没有依赖任何其他系统。

### 下游依赖（依赖此系统的系统）

| 系统 | 依赖类型 | 依赖内容 |
|------|----------|----------|
| **伤害计算系统** | 硬依赖 | 需要攻击力、护甲、稀有度加成 |
| **目标选择 AI** | 硬依赖 | 需要职业类型、攻击范围、移动速度 |
| **网格布局系统** | 硬依赖 | 需要单位 ID、名称、职业类型 |
| **克制系统** | 硬依赖 | 需要职业类型以查找克制关系 |
| **单位协同系统** | 软依赖 | 需要单位 ID 和职业类型检测组合 |
| **商店系统** | 硬依赖 | 需要价格、稀有度、单位 ID |
| **敌人系统** | 硬依赖 | 敌人使用相同的单位定义 |
| **布局编辑 UI** | 硬依赖 | 需要显示单位信息 |
| **商店 UI** | 硬依赖 | 需要显示商品信息 |

### 接口契约

此系统向所有下游系统承诺：

1. **数据完整性**：所有单位定义在游戏启动时加载完成
2. **ID 唯一性**：每个单位 ID 全局唯一
3. **属性有效性**：所有属性值在定义的范围内
4. **引用有效性**：技能引用在加载时验证

### 变更通知

如果此系统发生变更（如添加新属性），需要通知所有下游系统进行适配。

## Tuning Knobs

设计者可以调整的参数，用于游戏平衡。

### 稀有度系数

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|----------|------|
| `RARITY_RARE_MULTIPLIER` | 1.2 | 1.1 - 1.5 | 稀有单位强度 |
| `RARITY_EPIC_MULTIPLIER` | 1.5 | 1.3 - 2.0 | 史诗单位强度 |
| `RARITY_LEGENDARY_MULTIPLIER` | 2.0 | 1.5 - 3.0 | 传说单位强度 |

**风险**：
- 设置过高会破坏平衡，传说单位过于强大
- 设置过低会降低收集动机

### 价格倍率

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|----------|------|
| `RARITY_RARE_PRICE` | 2x | 1.5x - 3x | 稀有单位价格 |
| `RARITY_EPIC_PRICE` | 4x | 3x - 6x | 史诗单位价格 |
| `RARITY_LEGENDARY_PRICE` | 10x | 5x - 20x | 传说单位价格 |

**风险**：
- 设置过高会让稀有单位难以获取
- 设置过低会让收集进度过快

### 属性范围约束

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|----------|------|
| `MAX_HP` | 9999 | 1000 - 99999 | 单位最大生命值 |
| `MAX_ATTACK` | 999 | 100 - 9999 | 单位最大攻击力 |
| `MAX_ATTACK_SPEED` | 5.0 | 2.0 - 10.0 | 单位最大攻击速度 |
| `MAX_ARMOR` | 100 | 50 - 200 | 单位最大护甲 |

**风险**：
- 过高的上限可能导致数值膨胀
- 过低的上限可能限制设计空间

### 全局调节

| 旋钮 | 描述 | 用途 |
|------|------|------|
| `GLOBAL_STAT_MULTIPLIER` | 所有属性的全局乘数 | 快速调整游戏整体难度 |

**注意**：此旋钮应谨慎使用，仅在整体平衡需要调整时使用。

## Visual/Audio Requirements

### 视觉需求

| 需求 | 描述 |
|------|------|
| **单位图标** | 每个单位需要一个 32x32 或 64x64 像素图标 |
| **职业视觉区分** | 不同职业有明显的视觉特征（战士持剑、弓手持弓等） |
| **稀有度视觉指示** | 边框颜色或背景指示稀有度（灰、蓝、紫、金） |

### 音频需求

| 需求 | 描述 |
|------|------|
| **攻击音效** | 每个职业有独特的攻击音效 |
| **受击音效** | 通用的受击音效 |
| **死亡音效** | 单位死亡时播放 |

### 注意

- 视觉和音频资源由美术和音频团队定义
- 单位数据定义系统只提供资源引用字段
- 实际资源在资源系统中管理

## Acceptance Criteria

### 功能验收

- [ ] **AC1**: 系统能加载至少 5 个单位定义（MVP 范围）
- [ ] **AC2**: 每个单位定义包含所有必需属性（hp, attack, attack_speed, attack_range, armor, move_speed）
- [ ] **AC3**: 每个单位定义包含元数据（id, name, class_type, rarity, price）
- [ ] **AC4**: 稀有度加成正确应用到属性计算
- [ ] **AC5**: 技能引用能正确解析

### 数据验证

- [ ] **AC6**: 无效属性值（如 HP <= 0）触发验证错误
- [ ] **AC7**: 重复 ID 触发验证错误
- [ ] **AC8**: 无效技能引用触发验证错误
- [ ] **AC9**: 无效职业类型触发验证错误

### 接口验收

- [ ] **AC10**: 伤害计算系统能获取单位攻击力和护甲
- [ ] **AC11**: 目标选择 AI 能获取职业类型和攻击范围
- [ ] **AC12**: 商店系统能获取价格和稀有度
- [ ] **AC13**: UI 系统能获取显示名称和元数据

### 性能验收

- [ ] **AC14**: 加载 100 个单位定义耗时 < 100ms
- [ ] **AC15**: 查询单位定义耗时 < 1ms（缓存命中）
