# 自动战斗系统 (Auto Battle System)

> **Status**: Designed (pending review)
> **Author**: user + game-designer + ai-programmer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 策略深度（核心玩法的核心）

## Overview

自动战斗系统是游戏的核心引擎，负责执行所有单位的战斗行为。它协调单位移动、攻击选择、伤害应用和死亡处理。战斗自动进行，玩家只需观战。系统使用目标选择 AI 选择攻击目标，使用伤害计算系统计算伤害，最终决定战斗结果。

## Player Fantasy

这个系统服务于「策略深度」支柱的核心——玩家通过布局影响战斗，然后观看自己的策略在战场上展开。战斗应该是可视的、可理解的，让玩家能从每次战斗中学习和改进。观战本身就是一种享受。

## Detailed Design

### Core Rules

#### 1. 战斗时序模型

采用**回合制**：
- 战斗分为多个回合（Turn）
- 每回合内，所有单位按顺序行动
- 行动顺序由速度或位置决定（可选）
- 回合结束后，进入下一回合

#### 2. 单位行动流程

每回合每个单位执行：

```
1. 选择目标（目标选择 AI）
2. 如果目标不在范围内：
   - 移动向目标（如果可移动）
   - 结束行动
3. 如果目标在范围内：
   - 执行攻击
   - 计算伤害
   - 应用伤害
   - 检查目标死亡
4. 检查战斗结束条件
```

#### 3. 移动规则

| 规则 | 描述 |
|------|------|
| 移动速度 | 每回合最多移动 `move_speed` 格 |
| 移动路径 | 不能穿过其他单位 |
| 移动目标 | 向最近的目标移动 |
| 移动时机 | 攻击前移动 |

#### 4. 攻击规则

| 规则 | 描述 |
|------|------|
| 攻击间隔 | 基于 `attack_speed` 计算是否可以攻击 |
| 攻击范围 | `distance <= attack_range` |
| 攻击动画 | 攻击时有简短动画 |
| 攻击锁定 | 攻击时不能移动 |

#### 5. 回合顺序

```
回合顺序决定方式:
1. 按位置排序（从左上到右下）
2. 或者按速度排序（先攻值）
3. MVP 使用固定位置排序
```

#### 6. 战斗结束条件

| 条件 | 结果 |
|------|------|
| 敌方全灭 | 玩家胜利 |
| 我方全灭 | 玩家失败 |
| 超时（N 回合）| 玩家失败 |
| 超时（N 秒）| 玩家失败 |

### States and Transitions

每个单位的行动状态：

```
单位行动状态:
- IDLE（空闲）
- MOVING（移动中）
- ATTACKING（攻击中）
- DEAD（已死亡）

状态转换:
IDLE --选择移动--> MOVING
MOVING --移动完成--> IDLE
IDLE --选择攻击--> ATTACKING
ATTACKING --攻击完成--> IDLE
* --HP <= 0--> DEAD
```

## Formulas

### 回合时长

```
TURN_DURATION = 1.0  # 秒（每回合的游戏时间）
```

### 移动计算

```
移动步数 = min(move_speed, distance_to_target)
移动方向 = normalize(target_position - current_position)
新位置 = current_position + 移动方向 × 移动步数
```

### 攻击时机

```
can_attack = true  # 回合制中，每回合可攻击一次
# 或（如果使用攻击速度）
attack_cooldown -= TURN_DURATION
can_attack = attack_cooldown <= 0
```

### 战斗时长限制

```
MAX_TURNS = 100  # 最大回合数
MAX_TIME = 180   # 最大秒数
```

## Edge Cases

### 1. 移动碰撞

| 情况 | 处理方式 |
|------|----------|
| 目标格子被占用 | 移动到最近的空格子 |
| 移动路径被阻挡 | 绕行或停在障碍前 |
| 所有路径被阻挡 | 不移动 |

### 2. 目标问题

| 情况 | 处理方式 |
|------|----------|
| 目标死亡时正在移动 | 继续移动到目标位置，然后重新选择目标 |
| 目标死亡时正在攻击 | 取消攻击，重新选择目标 |
| 无目标可攻击 | 待机（IDLE）|

### 3. 同时死亡

| 情况 | 处理方式 |
|------|----------|
| 最后两个单位互相杀死对方 | 判定为失败（无获胜者）|
| 多个单位同时死亡 | 按顺序处理，检查结束条件 |

### 4. 战斗卡住

| 情况 | 处理方式 |
|------|----------|
| 所有单位无法互相攻击（距离太远且无法移动）| 判定为失败 |
| 战斗超过最大回合数 | 判定为失败 |
