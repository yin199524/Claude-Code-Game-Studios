# 目标选择 AI (Target Selection AI)

> **Status**: Designed (pending review)
> **Author**: user + game-designer + ai-programmer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 策略深度（目标选择影响战斗策略）

## Overview

目标选择 AI 系统负责决定每个单位在战斗中选择攻击哪个敌人。它根据单位的职业类型、攻击范围和战场位置，评估所有可能的目标并选择最优目标。这个系统让自动战斗具有策略性——玩家可以通过布局影响单位的攻击目标。

## Player Fantasy

这个系统服务于「策略深度」支柱。玩家应该能够预测单位的攻击目标，并根据这一点优化布局。例如，将高攻击力的单位放在能优先攻击敌方核心单位的位置。目标选择的可预测性让策略有意义。

## Detailed Design

### Core Rules

#### 1. 目标选择流程

```
输入: 攻击者单位, 所有敌方单位列表
输出: 目标单位（或 null 表示无目标）

流程:
1. 过滤：移除死亡、不在攻击范围内的敌方单位
2. 排序：按距离从近到远排序
3. 选择：返回第一个有效目标
```

#### 2. 距离计算

使用曼哈顿距离或欧几里得距离：

```
曼哈顿距离 = |x1 - x2| + |y1 - y2|
欧几里得距离 = sqrt((x1 - x2)² + (y1 - y2)²)
```

**MVP 使用曼哈顿距离**（网格游戏标准）

#### 3. 攻击范围判断

```
is_in_range = distance <= attack_range
```

- `attack_range = 1`：必须相邻（贴身近战）
- `attack_range = 2`：距离 2 格内
- `attack_range = 3+`：远程单位

#### 4. 职业特殊规则（可选扩展）

| 职业 | 特殊规则 |
|------|----------|
| 战士 | 无特殊 |
| 弓手 | 无特殊 |
| 法师 | 无特殊 |
| 骑士 | 无特殊 |
| 治疗 | 选择血量百分比最低的友方单位 |

**注意**：MVP 阶段所有攻击型单位使用"最近优先"规则，只有治疗单位例外。

#### 5. 目标切换规则

| 情况 | 行为 |
|------|------|
| 当前目标死亡 | 立即重新选择目标 |
| 当前目标移出范围 | 继续追击（如果可移动）或重新选择 |
| 新目标更近 | 不切换（保持当前目标，避免频繁切换） |

#### 6. 无目标情况

| 情况 | 处理 |
|------|------|
| 攻击范围内无敌人 | 如果可移动：向最近敌人移动<br>如果不可移动：等待 |
| 所有敌人已死亡 | 战斗结束 |

### States and Transitions

目标选择 AI 本身无状态，但单位的目标追踪有状态：

```
单位目标状态:
- current_target: UnitInstance | null
- target_lock_time: float  # 锁定目标的时间（用于避免频繁切换）
```

| 状态 | 转换条件 |
|------|----------|
| 无目标 | 攻击范围内无敌人 |
| 已锁定目标 | 选择目标后，锁定 until 目标死亡/移出范围 |
| 切换目标 | 目标死亡或移出范围后重新选择 |

## Formulas

### 距离公式

```
manhattan_distance(a, b) = |a.x - b.x| + |a.y - b.y|
euclidean_distance(a, b) = sqrt((a.x - b.x)² + (a.y - b.y)²)
```

**MVP 使用曼哈顿距离**

### 目标评分公式

用于在多个等距目标中选择（未来扩展用）：

```
target_score = base_score
             + distance_factor × (1 / distance)
             + hp_factor × (1 / target.hp_percent)
             + counter_factor × has_counter(attacker, target)
```

| 因子 | 当前值 | 说明 |
|------|--------|------|
| `base_score` | 100 | 基础分 |
| `distance_factor` | 50 | 距离权重 |
| `hp_factor` | 0 | MVP 不考虑血量 |
| `counter_factor` | 0 | MVP 不考虑克制 |

### 目标锁定时间

```
TARGET_LOCK_DURATION = 1.0  # 秒
```

锁定期间不切换目标（避免频繁切换影响观感）

## Edge Cases

### 1. 多个等距目标

| 情况 | 处理方式 |
|------|----------|
| 2+ 敌人距离相同 | 选择最先进入列表的（确定性顺序） |
| 空列表输入 | 返回 null |

### 2. 攻击范围边界

| 情况 | 处理方式 |
|------|----------|
| 敌人刚好在边界上 | 视为在范围内（distance <= range） |
| 攻击范围 = 0 | 只能攻击同一格的单位（通常不可能） |

### 3. 移动中的目标

| 情况 | 处理方式 |
|------|----------|
| 目标正在移动 | 使用当前位置计算距离 |
| 目标即将移出范围 | 当前攻击继续，下次选择时重新评估 |

### 4. 治疗单位特殊处理

| 情况 | 处理方式 |
|------|----------|
| 所有友方满血 | 无目标（待机） |
| 多个友方受伤 | 选择血量百分比最低的 |
| 自己受伤 | 可以治疗自己（取决于设计） |

## Acceptance Criteria

### 功能验收

- [ ] **AC1**: 正确选择最近的敌方单位
- [ ] **AC2**: 正确过滤攻击范围外的单位
- [ ] **AC3**: 目标死亡后正确切换到下一个目标
- [ ] **AC4**: 无目标时返回 null
- [ ] **AC5**: 治疗单位选择血量最低的友方

### 确定性验证

- [ ] **AC6**: 相同输入总是返回相同结果
- [ ] **AC7**: 多个等距目标的选择顺序一致

### 性能验收

- [ ] **AC8**: 单次目标选择耗时 < 0.1ms
- [ ] **AC9**: 100 个单位的目标选择总耗时 < 10ms
