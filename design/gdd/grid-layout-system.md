# 网格布局系统 (Grid Layout System)

> **Status**: Designed (pending review)
> **Author**: user + game-designer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 策略深度（布局是核心玩法）

## Overview

网格布局系统管理玩家在战斗开始前的单位放置。玩家在网格上拖放单位，每个单位占用一个格子。布局完成后，单位位置在战斗中不可更改。这个系统是核心玩法的入口——所有策略都从布局开始。

## Player Fantasy

这个系统服务于「策略深度」支柱。玩家应该感觉自己是军团的建筑师——每一个位置决策都有意义，不同的布局带来不同的战斗结果。布局过程应该是轻松的、可调整的，让玩家有时间思考和优化。

## Detailed Design

### Core Rules

#### 1. 网格配置

| 属性 | 值 |
|------|-----|
| 网格大小 | 3 × 3 = 9 格 |
| 格子索引 | (0,0) 到 (2,2)，左上角为原点 |
| 玩家区域 | 一侧（如最靠近玩家的 1-2 行）|
| 敌人区域 | 另一侧 |

#### 2. 布局限制

| 规则 | 描述 |
|------|------|
| 每格 1 单位 | 同一格子只能有一个单位 |
| 总单位限制 | 玩家最多放置 N 个单位（由关卡定义） |
| 敌人预置 | 敌人由关卡配置预先放置 |

#### 3. 布局操作

| 操作 | 描述 |
|------|------|
| 放置单位 | 从单位列表拖放到网格 |
| 移动单位 | 拖动已放置的单位到其他格子 |
| 移除单位 | 拖动单位回单位列表 |
| 清空布局 | 一键移除所有单位 |

#### 4. 布局验证

| 检查 | 结果 |
|------|------|
| 格子已有单位 | 拒绝放置，提示"格子已被占用" |
| 单位数量达到上限 | 拒绝放置，提示"已达到单位上限" |
| 格子在敌人区域 | 拒绝放置，提示"不能放置在此区域" |

#### 5. 布局保存

- 布局完成后点击"开始战斗"按钮确认
- 确认后布局锁定，进入战斗状态
- 可以为每个关卡保存多个布局预设（后期功能）

#### 6. 坐标系

```
网格坐标系:
  ┌─────────────────┐
  │ (0,0) (1,0) (2,0) │ ← 敌人区域（假设）
  │ (0,1) (1,1) (2,1) │
  │ (0,2) (1,2) (2,2) │ ← 玩家区域（假设）
  └─────────────────┘
```

### States and Transitions

```
布局状态机:

[空闲] --选择关卡--> [布局中]
[布局中] --确认布局--> [已确认]
[已确认] --战斗结束--> [空闲]
[布局中] --取消--> [空闲]
```

| 状态 | 描述 | 允许操作 |
|------|------|----------|
| **空闲** | 未开始布局 | 选择关卡 |
| **布局中** | 正在放置单位 | 放置、移动、移除单位 |
| **已确认** | 布局完成，战斗中 | 无（布局锁定）|

## Formulas

### 位置计算

```
网格坐标 → 屏幕坐标:
screen_x = grid_origin_x + cell_size × grid_x
screen_y = grid_origin_y + cell_size × grid_y

屏幕坐标 → 网格坐标:
grid_x = floor((screen_x - grid_origin_x) / cell_size)
grid_y = floor((screen_y - grid_origin_y) / cell_size)
```

### 距离计算（用于 UI）

```
曼哈顿距离 = |x1 - x2| + |y1 - y2|
```

### 布局完成度

```
layout_progress = placed_units / max_units × 100%
```

## Edge Cases

### 1. 布局为空

| 情况 | 处理方式 |
|------|----------|
| 玩家未放置任何单位 | 显示提示"请至少放置一个单位" |
| 点击"开始战斗"按钮 | 拒绝，要求至少放置 1 个单位 |

### 2. 格子边界

| 情况 | 处理方式 |
|------|----------|
| 拖放到网格外 | 单位回到原位置 |
| 拖放到边界上 | 判断格子中心，决定归属 |

### 3. 快速操作

| 情况 | 处理方式 |
|------|----------|
| 快速连续放置 | 按顺序处理，不丢失操作 |
| 放置动画未完成时移动 | 取消动画，直接更新位置 |

### 4. 网络延迟（如果有多人）

| 情况 | 处理方式 |
|------|----------|
| 同一格子被两人放置 | 先到先得，后到者收到错误提示 |

## Acceptance Criteria

### 功能验收

- [ ] **AC1**: 可以从单位列表拖放单位到网格
- [ ] **AC2**: 可以移动已放置的单位
- [ ] **AC3**: 可以移除已放置的单位
- [ ] **AC4**: 同一格子只能有一个单位
- [ ] **AC5**: 不能放置在敌人区域
- [ ] **AC6**: 单位数量达到上限时拒绝放置
- [ ] **AC7**: 布局为空时不能开始战斗

### 交互验收

- [ ] **AC8**: 拖动时显示有效/无效位置指示
- [ ] **AC9**: 释放时自动对齐到格子中心
- [ ] **AC10**: 操作有视觉反馈

### 性能验收

- [ ] **AC11**: 拖动操作响应延迟 < 50ms
- [ ] **AC12**: 布局数据保存/加载 < 10ms
