# 成就系统设计文档

## Overview

成就系统为玩家提供长期目标和奖励，通过记录和展示玩家在游戏中的各种里程碑，增强游戏的重玩价值和满足感。系统支持多种成就类型，包括进度、收集、战斗、升级和协同等维度。

## Player Fantasy

**"我的每一次努力都被铭记，每一个里程碑都值得庆祝。"**

玩家希望：
- 看到自己在游戏中的成长和进步
- 获得完成目标的满足感和奖励
- 与其他玩家分享自己的成就
- 有明确的长期目标去追求

## Detailed Rules

### 成就解锁机制

1. **实时检测**: 成就条件在游戏中实时检测
2. **自动解锁**: 条件满足时自动解锁成就
3. **一次性**: 每个成就只能解锁一次
4. **奖励发放**: 解锁时立即发放奖励

### 成就类别

| 类别 | 触发时机 | 检测方式 |
|------|----------|----------|
| PROGRESS | 关卡完成时 | LevelManager 触发 |
| COLLECTION | 单位获取时 | ShopSystem 触发 |
| COMBAT | 战斗结算时 | BattleManager 触发 |
| UPGRADE | 升级完成时 | UpgradeSystem 触发 |
| SYNERGY | 战斗开始时 | SynergyManager 触发 |
| SPECIAL | 特定条件 | 各系统触发 |

### 进度追踪

- 每个成就维护当前进度值
- 进度实时更新，不需要重新计算
- 已达成的成就不再更新进度

## Formulas

### 成就完成率

```
完成率 = 已解锁成就数 / 总成就数
```

### 奖励计算

成就奖励为固定值，设计时考虑：
- 早期成就: 50-100 金币（引导性质）
- 中期成就: 200-500 金币（挑战性质）
- 后期成就: 500-1000 金币（高难度）

## Edge Cases

### 离线期间达成

- 成就检测依赖事件触发
- 离线期间不会发生需要检测的事件
- 无需特殊处理

### 重复触发

- 已解锁成就忽略后续触发
- 进度不再更新
- 奖励只发放一次

### 成就依赖

- 某些成就可能有隐性依赖（如通关沙漠需要先通关森林）
- 依赖由游戏流程保证，成就系统不做强制检查

## Dependencies

- **SaveManager**: 持久化成就进度
- **EventBus**: 跨系统事件通信
- **LevelManager**: 进度类成就检测
- **ShopSystem**: 收集类成就检测
- **BattleManager**: 战斗类成就检测
- **UpgradeSystem**: 升级类成就检测
- **SynergyManager**: 协同类成就检测

## Tuning Knobs

| 参数 | 默认值 | 说明 |
|------|--------|------|
| ACHIEVEMENT_COUNT | 14 | V1 版本成就数量 |
| REWARD_EARLY_ACHIEVEMENT | 100 | 早期成就奖励 |
| REWARD_MID_ACHIEVEMENT | 300 | 中期成就奖励 |
| REWARD_LATE_ACHIEVEMENT | 1000 | 后期成就奖励 |

## Acceptance Criteria

- [ ] 玩家可以在成就面板查看所有成就
- [ ] 成就按类别分组显示
- [ ] 已解锁成就显示解锁时间
- [ ] 未解锁成就显示进度百分比
- [ ] 成就解锁时有视觉特效
- [ ] 成就解锁时有音效反馈
- [ ] 成就进度正确持久化
- [ ] 游戏重启后进度不丢失
- [ ] 所有14个成就可正常解锁
- [ ] 奖励正确发放到玩家账户
