# 伤害计算系统 (Damage Calculation System)

> **Status**: Designed (pending review)
> **Author**: user + game-designer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 策略深度（伤害公式影响战斗策略）

## Overview

伤害计算系统负责计算战斗中单位之间的伤害输出。它接收攻击者和防御者的属性，应用护甲减伤和克制加成，输出最终伤害值。这是战斗系统的核心计算引擎，决定了战斗的节奏和策略深度。

## Player Fantasy

这个系统服务于「策略深度」支柱。玩家应该能够通过理解伤害公式来优化布局——将克制敌人的单位放在合适的位置，最大化伤害输出。伤害数字的可预测性让策略有意义，而不是纯粹的随机。

## Detailed Design

### Core Rules

#### 1. 伤害计算流程

```
输入: 攻击者单位, 防御者单位
输出: 最终伤害值

流程:
1. 获取攻击者攻击力（应用稀有度加成）
2. 获取防御者护甲
3. 计算护甲减伤
4. 应用克制加成（如果适用）
5. 应用随机波动
6. 输出最终伤害（最小为 1）
```

#### 2. 护甲减伤规则

采用**混合模型**：
- 百分比减伤：护甲越高，减伤比例越高
- 固定减伤：护甲提供固定数值减免

#### 3. 克制加成规则

- 当攻击者职业克制防御者职业时，伤害 × 1.2
- 克制关系由克制系统定义
- MVP 阶段克制关系可在单位数据中硬编码

#### 4. 随机性规则

- 最终伤害有 ±10% 的随机波动
- 波动在每次攻击时重新计算
- 使用确定性随机种子（便于回放）

### States and Transitions

**此系统无运行时状态。** 伤害计算是纯函数，输入相同则输出相同（忽略随机性）。

### Interactions with Other Systems

#### 输入接口

| 来源系统 | 数据 | 用途 |
|----------|------|------|
| **单位数据定义** | attack, armor, rarity, class_type | 基础属性 |
| **克制系统** | 克制关系表 | 判断是否有克制加成 |
| **战斗系统** | 攻击者实例, 防御者实例 | 触发伤害计算 |

#### 输出接口

| 目标系统 | 数据 | 用途 |
|----------|------|------|
| **自动战斗系统** | 最终伤害值 | 扣减 HP |
| **战斗 UI** | 伤害数值 | 显示伤害数字 |

## Formulas

### 核心伤害公式

```
base_damage = attacker.attack × attacker.rarity_multiplier
armor_percent_reduction = armor / (armor + ARMOR_SCALING_CONSTANT)
armor_flat_reduction = armor × ARMOR_FLAT_RATIO
pre_random_damage = base_damage × (1 - armor_percent_reduction) - armor_flat_reduction
counter_multiplier = has_counter ? COUNTER_BONUS : 1.0
random_factor = random_range(0.9, 1.1)
final_damage = max(1, floor(pre_random_damage × counter_multiplier × random_factor))
```

### 常量定义

| 常量 | 值 | 说明 |
|------|-----|------|
| `ARMOR_SCALING_CONSTANT` | 100 | 护甲百分比减伤缩放常数 |
| `ARMOR_FLAT_RATIO` | 0.2 | 护甲固定减伤比例 |
| `COUNTER_BONUS` | 1.2 | 克制加成倍率 |
| `RANDOM_RANGE` | 0.9 ~ 1.1 | 随机波动范围 |
| `MIN_DAMAGE` | 1 | 最小伤害 |

### 护甲减伤计算示例

| 护甲 | 百分比减伤 | 固定减伤 | 总减伤（对 100 攻击） |
|------|-----------|----------|----------------------|
| 0 | 0% | 0 | 0 |
| 20 | 16.7% | 4 | ~21 |
| 50 | 33.3% | 10 | ~43 |
| 100 | 50% | 20 | ~70 |

### 克制加成示例

假设基础伤害 100，无护甲：
- 无克制：伤害 = 100 × (0.9~1.1) = 90~110
- 有克制：伤害 = 100 × 1.2 × (0.9~1.1) = 108~132

### 稀有度影响

攻击力应用稀有度加成（来自单位数据定义）：

| 稀有度 | 攻击力乘数 |
|--------|-----------|
| Common | 1.0x |
| Rare | 1.2x |
| Epic | 1.5x |
| Legendary | 2.0x |

## Edge Cases

### 1. 伤害边界

| 情况 | 处理方式 |
|------|----------|
| 伤害计算结果 < 1 | 强制为 1（MIN_DAMAGE） |
| 攻击力 = 0 | 伤害 = 1（最小伤害） |
| 护甲 > 攻击力 | 伤害 = 1（最小伤害） |
| 护甲极大（如 999） | 伤害 = 1（百分比减伤有上限） |

### 2. 克制关系缺失

| 情况 | 处理方式 |
|------|----------|
| 克制系统未设计 | 使用硬编码克制关系或无克制 |
| 职业类型不在克制表中 | 视为无克制关系 |
| 双向克制 | 攻击者和防御者都获得加成？待定 |

### 3. 随机种子

| 情况 | 处理方式 |
|------|----------|
| 战斗回放 | 使用相同的随机种子，结果可重现 |
| 网络同步 | 随机种子由主机生成并同步 |

### 4. 数值溢出

| 情况 | 处理方式 |
|------|----------|
| 攻击力超过 int 范围 | 数据验证限制最大值 |
| 伤害计算溢出 | 使用 64 位整数或浮点数 |

## Tuning Knobs

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|----------|------|
| `ARMOR_SCALING_CONSTANT` | 100 | 50 - 200 | 护甲百分比减伤效率 |
| `ARMOR_FLAT_RATIO` | 0.2 | 0.1 - 0.5 | 护甲固定减伤比例 |
| `COUNTER_BONUS` | 1.2 | 1.1 - 2.0 | 克制伤害加成 |
| `RANDOM_RANGE_MIN` | 0.9 | 0.8 - 1.0 | 随机波动下限 |
| `RANDOM_RANGE_MAX` | 1.1 | 1.0 - 1.3 | 随机波动上限 |
| `MIN_DAMAGE` | 1 | 1 - 10 | 最小伤害值 |

### 调节建议

- **提高护甲价值**：增加 `ARMOR_SCALING_CONSTANT`
- **加快战斗节奏**：降低 `ARMOR_FLAT_RATIO` 或提高 `MIN_DAMAGE`
- **增强策略性**：增加 `COUNTER_BONUS`
- **降低随机性**：缩小 `RANDOM_RANGE`

## Acceptance Criteria

### 功能验收

- [ ] **AC1**: 给定相同输入，伤害计算结果一致（除随机波动）
- [ ] **AC2**: 护甲正确减少伤害
- [ ] **AC3**: 克制关系正确应用 20% 加成
- [ ] **AC4**: 随机波动在 ±10% 范围内
- [ ] **AC5**: 最小伤害为 1

### 公式验证

- [ ] **AC6**: 0 护甲目标受到全额伤害（含随机波动）
- [ ] **AC7**: 100 护甲目标受到约 50% 减少
- [ ] **AC8**: 克制加成与其他乘数正确叠加

### 性能验收

- [ ] **AC9**: 单次伤害计算耗时 < 0.1ms
- [ ] **AC10**: 100 次计算耗时 < 10ms
