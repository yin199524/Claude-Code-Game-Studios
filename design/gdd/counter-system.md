# 克制系统 (Counter System)

> **Status**: Designed
> **Author**: game-designer
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 策略深度

## Overview

克制系统定义了不同职业之间的相克关系，增加战斗策略深度。玩家需要考虑敌方单位职业，选择有优势的单位进行对抗。克制关系影响伤害输出，鼓励玩家思考布局策略。

## Player Fantasy

玩家在布局阶段思考"我的战士能克制他的骑士吗？"、"对面有法师，我应该上弓手"，体验策略游戏的深度思考乐趣。克制系统让每一次单位选择都有意义，而不是随机放置。

## Detailed Design

### Core Rules

1. **克制关系是有向的**：A 克制 B ≠ B 克制 A
2. **克制影响伤害**：克制方伤害 +30%，被克制方伤害 -20%
3. **克制关系不影响攻击速度**：只影响最终伤害值
4. **克制只影响玩家单位与敌人单位之间的互动**
5. **同职业不存在克制**：伤害按正常计算

### 职业克制关系表

| 职业 | 克制 | 被克制 |
|------|------|--------|
| 战士 (WARRIOR) | 骑士 | 弓手 |
| 弓手 (ARCHER) | 法师、治疗 | 战士 |
| 法师 (MAGE) | 骑士 | 弓手 |
| 骑士 (KNIGHT) | 治疗 | 战士、法师 |
| 治疗 (HEALER) | 无 | 骑士 |

### 克制链图示

```
战士 → 骑士 → 治疗
  ↑      ↓
弓手 ← 法师
  ↓
(克制战士)
```

### States and Transitions

克制计算发生在伤害计算阶段：

```
攻击发起 → 检查双方职业 → 查克制表 → 计算克制倍率 → 应用到伤害
```

## Formulas

### 克制伤害公式

```
final_damage = base_damage × counter_multiplier

其中：
- base_damage = 原始伤害（已应用护甲减免）
- counter_multiplier:
  - 克制方: 1.3 (+30%)
  - 被克制方: 0.8 (-20%)
  - 无克制: 1.0
```

### 示例计算

战士 (攻击力 40) 攻击骑士：
- 基础伤害: 40
- 战士克制骑士: ×1.3
- 最终伤害: 40 × 1.3 = 52

骑士 (攻击力 55) 攻击战士：
- 基础伤害: 55
- 骑士被战士克制: ×0.8
- 最终伤害: 55 × 0.8 = 44

## Edge Cases

1. **同职业对战**: 不触发克制，伤害 ×1.0
2. **治疗职业**: 治疗不克制任何职业，但被骑士克制
3. **治疗技能**: 治疗效果不受克制影响（只影响伤害）
4. **多对多**: 每次攻击独立计算克制，不叠加

## Dependencies

- **上游**: 单位数据定义 (UnitDefinition.class_type)
- **下游**: 伤害计算系统、战斗 UI

## Tuning Knobs

| 参数 | 默认值 | 范围 | 影响 |
|------|--------|------|------|
| 克制倍率 | 1.3 | 1.1 - 1.5 | 克制方伤害加成 |
| 被克制倍率 | 0.8 | 0.6 - 0.9 | 被克制方伤害减少 |

## Visual/Audio Requirements

- 克制攻击显示特殊颜色伤害数字（金色）
- 被克制攻击显示削弱颜色（灰色）
- UI 中显示克制关系图标

## UI Requirements

- 商店/布局阶段显示克制关系提示
- 战斗中显示克制加成数值
- 单位信息面板显示克制/被克制职业
