# 单位协同系统 (Synergy System)

> **Status**: Designed
> **Author**: game-designer
> **Last Updated**: 2026-03-26
> **Implements Pillar**: 策略深度（队伍构建）

## Overview

协同系统让特定单位组合获得额外加成，鼓励玩家探索不同的队伍构建策略。当玩家在队伍中满足特定条件时，相关单位会获得属性加成，增加游戏的策略深度。

## Player Fantasy

玩家在组建队伍时思考——"如果我带两个战士，他们会形成战士兄弟情，一起战斗更勇猛！"这种协同感让队伍构建变得有意义，而不是简单地堆叠最强单位。

## Detailed Design

### Core Rules

1. **协同类型**: 三种协同类型
   - 职业协同：同职业单位数量达标
   - 阵型协同：特定位置配置
   - 混合协同：特定单位组合
2. **效果叠加**: 协同效果可叠加，同一单位可受多个协同加成
3. **触发时机**: 战斗开始时检测并应用
4. **视觉反馈**: 满足条件时显示协同激活提示

#### 阵型协同

| 名称 | 条件 | 效果 | 描述 |
|------|------|------|------|

| 名称 | 条件 | 效果 | 描述 |
|------|------|------|------|

| 魔武双修 | 队伍同时有法师+战士 | 法师+10% 伤害，战士+10% 攻击 | 魔法与武力的完美结合 |

### 协同激活流程

```
战斗开始 → 检测所有协同条件 → 计算每个单位获得的加成 → 应用加成 → 显示激活提示 → 战斗进行
```

## Formulas

### 属性加成计算

```
effective_stat = base_stat × (1 + synergy_bonus_1 + synergy_bonus_2 + ...)

其中:
- synergy_bonus_i: 第 i 个协同效果的加成比例
- 加成可叠加，无上限
```

## Edge Cases

### 1. 空队伍

| 情况 | 处理方式 |
|------|----------|
| 无单位 | 无协同激活 |
| 1个单位 | 检测但不满足任何条件 |

### 2. 多重协同

| 情况 | 处理方式 |
|------|----------|
| 同一单位受多个协同 | 效果叠加计算 |
| 协同效果超过 100% 加成 | 允许，无上限 |

### 3. 单位死亡

| 情况 | 处理方式 |
|------|----------|
| 协同单位死亡 | 协同效果保持（战斗开始时锁定） |
| 全部协同单位死亡 | 已应用的效果不变 |

## Tuning Knobs

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|----------|------|
| `SYNERGY_CLASS_THRESHOLD` | 2 | 2-4 | 触发职业协同所需数量 |
| `SYNERGY_BONUS_ATTACK` | 0.1 (10%) | 0.05-0.3 | 攻击加成比例 |
| `SYNERGY_BONUS_DEFENSE` | 0.1 (10%) | 0.05-0.3 | 防御加成比例 |
| `SYNERGY_BONUS_DAMAGE` | 0.15 (15%) | 0.1-0.25 | 伤害加成比例 |

### 调节建议

- **降低触发门槛**: 减小 `SYNERGY_CLASS_THRESHOLD` 让协同更容易触发
- **增加策略性**: 增加 `SYNERGY_BONUS_*` 让协同更有价值
- **鼓励多样性**: 添加更多混合协同类型
